# Hanbok Sales

**개인 프로젝트**  


---

## 한 일

**한복 판매 사이트의 모든 페이지를 설계하고 구현했습니다.**

* **페이지 설계**: 메인, 상품, 상세, 갤러리, 장바구니, 결제, 정보 페이지 구성
* **HTML/CSS/JavaScript**: 전체 페이지 개발
* **반응형 디자인**: 모바일부터 데스크톱까지 최적화
* **동적 기능**: 장바구니, 리뷰, Q&A 등 사용자 상호작용 구현
* **LocalStorage**: 브라우저에 데이터 저장 및 관리

**결과**: 완전한 프론트엔드 이커머스 웹사이트 완성

---

## 주요 기능

### 상품 탐색 및 필터링

* 상품 유형별 분류 (데일리 생활 한복, 명절 맞이 한복, 혼주 및 예복 한복)
* 가격대별 필터링
* 목록 및 그리드 레이아웃 지원

### 상품 상세 페이지

* 고화질 상품 이미지
* 상세 설명 및 사이즈 선택
* 색상 스와치(Swatch) 선택으로 실시간 이미지 변경
* 리뷰 및 Q&A 섹션
* 장바구니 추가 기능

### 리뷰 및 Q&A

* 상품별 리뷰 작성 (별점, 제목, 내용)
* 리뷰 목록 표시
* Q&A 섹션으로 고객 문의 답변
* LocalStorage에 데이터 저장

### 장바구니 기능

* 장바구니에 담긴 상품 목록 표시
* 수량 변경 기능
* 상품 삭제 기능
* 총 가격 자동 계산
* 헤더의 장바구니 카운터 뱃지 (실시간 업데이트)

### 주문 및 결제 페이지

* 주문자 정보 입력 폼
* 배송지 관리 (기본 배송지, 새로운 배송지 추가)
* 결제 정보 입력
* 주문 확인 및 완료 화면

### 반응형 디자인

* 모바일 환경 최적화
* 헤더 네비게이션은 작은 화면에서 햄버거 메뉴로 전환
* 모든 페이지가 모바일/태블릿/PC에서 최적으로 작동

## 배운 점

### 1. HTML/CSS로 반응형 디자인 구현

미디어 쿼리와 플렉스박스를 활용해 모바일부터 데스크톱까지 모든 화면 크기에서 최적의 레이아웃을 만들었습니다.

### 2. LocalStorage로 데이터 관리

장바구니, 리뷰, Q&A 등의 데이터를 LocalStorage에 JSON 형식으로 저장해 브라우저 새로고침 후에도 데이터가 유지되도록 구현했습니다.

### 3. 동적 UI 업데이트

사용자의 액션(상품 추가, 수량 변경, 상품 삭제)에 따라 DOM을 조작해 즉시 화면을 업데이트했습니다.

### 4. 폼 유효성 검사

주문 페이지에서 사용자 입력값을 검증하고 필수 항목을 확인하는 로직을 구현했습니다.

### 5. 사용자 경험 개선

장바구니 카운터 뱃지, 색상 선택 시 실시간 이미지 변경, 배경 이미지 최적화 등으로 사용자 경험을 향상시켰습니다.

### 6. 모바일 네비게이션

작은 화면에서 헤더 메뉴가 넘치지 않도록 햄버거 메뉴로 토글되는 네비게이션을 구현했습니다.
